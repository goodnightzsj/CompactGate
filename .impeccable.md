## Design Context

### Users
CompactGate is used by developers and local AI tooling operators who need to monitor and control an OpenAI-compatible reverse proxy, routing rules, compact request behavior, provider settings, health, logs, and saved configuration profiles during active development sessions.

### Brand Personality
Calm, precise, operator-grade. The interface should feel like a trustworthy local control room: dense enough for power users, but not intimidating or noisy.

### Aesthetic Direction
Use a refined technical console style with warm paper neutrals, ink-like typography, crisp borders, purposeful status colors, and measured density. Avoid generic AI-dashboard clichés: purple/cyan glow, excessive glassmorphism, decorative metric cards, random gradients, and noisy neon dark mode.

### Design Principles
1. Prioritize scanability: operators should see health, routing, and save/profile state quickly.
2. Keep controls tactile and explicit: buttons, inputs, selects, and destructive actions need clear affordance and focus states.
3. Use color semantically: green for healthy/success, amber for warning/pending, red for errors/destructive actions, blue/slate for informational routing.
4. Maintain compact rhythm without crowding: group related controls tightly and separate sections generously.
5. Design for real local usage: long model names, long profile names, empty logs, narrow windows, and repeated save/restart cycles must remain readable.
